# v0.5.6 — April 14, 2026

## New Features

### iOS Support
- **Gloam runs on iOS**: First cut of the iOS build — sign in, sync, chat, send/receive messages, and get notifications on your phone.
- **Home-screen badge**: Unread count shows on the app icon via `UNUserNotificationCenter.setBadgeCount` (iOS 16+).
- **Push permission flow**: First-run prompts for alerts/badges/sound on iOS alongside the existing macOS/Windows/Linux paths.
- **Test notification on iOS**: Settings → Send Test works on iOS too.

### Mobile Shell Rebuild
- **Three tabs instead of four**: Replaced the `chats | spaces | calls | settings` placeholder layout with a focused `chats | inbox | me` layout. Spaces now live as a horizontal switcher rail inside Chats; Calls surface in the persistent voice bar only when active; Settings moves under Me.
- **Chats tab**: Logo header with search, horizontal space switcher rail under it, room list below, and a FAB bottom-right for starting a new conversation (new DM, new room; space creation remains desktop-only for now).
- **Inbox tab**: Surfaces the things that need your attention — pending room/space invites (actionable inline) and rooms with unread mentions.
- **Me tab**: Profile header + settings access in one place.

## Bug Fixes

### Code-Block Rendering
- **Multi-line fenced blocks with inline backticks render correctly**: Pasting a diff or code sample that itself contains ` ``` ` used to make CommonMark reject the outer fence, collapsing everything into a paragraph of inline code with newlines merged to spaces. When the whole message is a ``` -wrapped block, Gloam now reconstructs the code block from the plain-text body and preserves every line break.
- **Single-line ` ```text``` ` no longer shows as a blank box**: When the user typed a one-line triple-backtick wrap, CommonMark was parsing the text as the info string (language metadata) with an empty body. Gloam now folds that info string back in as content — Slack/Discord-style.
- **Prose first lines inside code blocks are preserved**: Messages like ```` ```⏺ Root cause: ... ```` where the first line wasn't a valid language token were losing that first line to the `language-`/`data-metadata` attributes. The fold logic now detects non-language tokens and restores them as the first line of code.
- **Newlines inside attributed `<pre>` tags are preserved**: Worked around an upstream Matrix SDK bug (`convertLinebreaksToBr` splits on the literal string `pre>`, which doesn't match `<pre data-metadata="…">`) that was silently deleting newlines inside code blocks whose `<pre>` carried attributes.

### `$` Is Just Text Now
- **`$9/$10` no longer renders as math**: The Matrix SDK's inline-LaTeX syntax was wrapping `$…$` as `<span data-mx-maths>`, which rendered as monospace code with the dollars swallowed. Gloam now escapes `$` at send time (outside code blocks and inline code) so recipients on any client see literal text, and unwraps any inbound LaTeX spans at render time so existing messages display correctly.
