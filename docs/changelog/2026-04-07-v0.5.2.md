# v0.5.2 — April 7, 2026

## New Features

### Encryption Bootstrap
- **Set up encryption wizard**: a guided modal that creates SSSS keys, cross-signing keys, and enables key backup — all in one flow. Drives the Matrix SDK's `Bootstrap` state machine automatically.
- **Recovery key presentation**: displays the Base58 recovery key with both "Copy to clipboard" and "Save to file" options. Confirmation checkbox required before continuing.
- **Sidebar banner**: accent-colored prompt at the top of the room list when encryption isn't set up. "Set up encryption to secure your messages" with a "Set up" button. Dismissible, auto-hides when bootstrap completes.
- **Settings entry point**: "Set up encryption" row in Settings > Security & Keys when cross-signing isn't enabled.

### Space Invites
- **Space invites in the space rail**: pending space invites now appear above joined spaces with a dashed border and accent dot badge. Tapping shows a rich invite card in the room list panel.
- **Rich invite card**: large space avatar, name, topic, inviter row (avatar + "invited you"), member count, and a best-effort channel preview list via the space hierarchy endpoint.
- **Accept/Decline**: prominent "Join Space" button and subtle "Decline" text link. Accepting keeps the space selected so the channel list loads immediately. Declining resets to the DMs view.

### Undecryptable Message UX
- **Pre-join vs post-join classification**: encrypted messages are now classified based on whether they were sent before or after the user joined the room.
- **Pre-join messages**: compact inline indicator — "Encrypted — sent before you joined" in muted text. When a key backup exists but the recovery key hasn't been entered, shows "Encrypted — enter your recovery key to view" with a tappable link to the recovery key dialog.
- **Post-join messages**: compact inline indicator — "Unable to decrypt" in warning color with a "Request key" action that silently fires a key request and changes to "Key requested..."

## Bug Fixes

### Room Invite Visibility
- **Room invites no longer hidden by space/category filters**: invites are now exempt from space filtering and unread/mentions filtering, ensuring they always appear in the room list regardless of the active view.

### Space Hierarchy Cache
- **Space hierarchy auto-refreshes on child changes**: the space hierarchy provider now watches for `m.space.child` state event changes and refetches when rooms are added or removed from a space. Previously, newly created channels wouldn't appear under their space until app restart.

### Recovery Key Dialog Crash
- **Fixed defunct element crash during recovery key entry**: the recovery dialog no longer invalidates all timeline providers (which caused a `StateNotifierProviderElement.create` crash on defunct widget elements). Instead, it calls `retryDecryption()` on active timeline notifiers to re-attempt decryption in place.
